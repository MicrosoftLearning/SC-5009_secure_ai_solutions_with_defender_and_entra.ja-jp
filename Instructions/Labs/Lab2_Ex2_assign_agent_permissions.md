# ラボ 2 - 演習 2 - AI エージェントのアクセス許可を構成する

組織の AI エージェントは、Microsoft Entra ID に登録されている **Contoso-AI-Agent** という名前のアプリケーション ID を使用します。 エージェントは、Azure AI サービスにアクセスし、Key Vault からシークレットを読み取り、Microsoft Graph からユーザー データをクエリする必要がありますが、登録された ID には既定ではアクセス許可がありません。 ID 管理者は、API アクセス許可と Azure RBAC ロールの両方を構成して、エージェントに必要なアクセス権のみを付与します。 ユーザーまたはグループへのアクセス権の付与とは異なり、ユーザーがサインインしなくても、独立して実行されるアプリケーション ID へのアクセス権を付与しています。

> **注**: アプリケーション ID では、2 種類のアクセス許可が使用されます。 **API のアクセス許可**、Microsoft Graph などの API を通じてデータへのアクセスを制御します。 **Azure RBAC ロール**、Foundry や Key Vault などの Azure リソースへのアクセスを制御します。 ほとんどの AI エージェントには両方が必要です。

**タスク**:

1. AI エージェントの API アクセス許可を追加する
1. AI エージェントに Foundry リソースへのアクセス権を付与する
1. AI エージェントに Key Vault へのアクセス権を付与する

## タスク 1 – AI エージェントの API アクセス許可を追加する

AI エージェントは、応答をカスタマイズするために、Microsoft Graph からユーザー プロファイル データを読み取る必要があります。 このタスクでは、エージェントのアプリ登録にアプリケーションレベルの API アクセス許可を追加します。これにより、対話型のサインインなしでユーザー データをクエリできるようになります。

1. **Microsoft Edge** を開き、`https://entra.microsoft.com/` に移動します。

1. ラボ ホスト プロバイダーによって提供されるユーザー名とパスワードを使用してサインインします。

1. 左側のサイド バーの **[Entra ID]** で、**[アプリの登録]** を選択します。

1. **Contoso-AI-Agent** を選択します。

1. 左側のサイド バーの **[管理]** で、**[API のアクセス許可]** を選択します。

1. **[+ アクセス許可の追加]** を選択します。

1. **[API アクセス許可の要求]** ポップアップで、**[Microsoft Graph]** を選択します。

1. **[アプリケーションのアクセス許可]** を選択します。

1. 検索フィールドに「`User.Read.All`」と入力し、**[User.Read.All]** アクセス許可チェックボックスをオンにします。

1. **アクセス許可の追加** を選択します。

1. **[API のアクセス許可]** ページに戻り、アクセス許可の一覧を確認します。 **[Microsoft Graph]** の下に **[User.Read.All]** が表示され、状態は **[未付与]** になっています。

    > **注**: このアクセス許可は、エージェントが使用する前に管理者の同意が必要です。 運用環境では、全体管理者または特権ロール管理者が確認し、同意を付与します。 "未付与" 状態は、アクセス許可が構成されているがまだアクティブでないことを意味します。これは、アプリケーションが明示的な承認なしでデータにアクセスできないようにする重要なセキュリティ チェックポイントです。

AI エージェントの API アクセス許可が正常に構成されました。 この時点で、エージェントには Microsoft Graph 経由でデータにアクセスするためのアクセス許可が構成されていますが、RBAC ロールが割り当てられるまで Azure リソースにはアクセスできません。

## タスク 2 – AI エージェントに Foundry リソースへのアクセス権を付与する

AI エージェントは、Foundry リソースでホストされている Azure AI サービスを呼び出す必要があります。 このタスクでは、エージェントのサービス プリンシパルに Cognitive Services ユーザー ロールを割り当てます。これにより、管理アクセス権を付与せずに推論呼び出しを行うことができます。

1. 新しいタブを開き、`https://portal.azure.com` に移動します。

1. **[リソース グループ]** を選択し、**rg-foundry-lab** リソース グループを選択します。

1. リソース グループで、種類が **Foundry** のリソースを選択します。

1. 左パネルから **[アクセスの制御 (IAM)]** を選択します。

1. **[+ 追加]** > **[ロールの割り当ての追加]** の順に選択します。

1. **[ロールの割り当ての追加]** ページの **[ジョブ関数ロール]** タブで、`Cognitive Services User` を検索して選択します。

1. [**次へ**] を選択します。

1. **[メンバー]** タブで、**[ユーザー、グループ、またはサービス プリンシパル]** オプションを選択してから、**[+ メンバーの選択]** を選択します。

1. **[メンバーの選択]** ポップアップで、`Contoso-AI-Agent` を検索し、アプリケーションを選択してから、**[選択]** を選択します。

1. **[次へ]** を選択し、**[確認と割り当て]** タブで、**[確認と割り当て]** を選択します。

1. ロールの割り当てが正常に追加されたことを示す通知が表示されます。

Foundry リソース上の Azure AI サービスへの AI エージェント ID アクセス権が正常に付与されました。

## タスク 3 – AI エージェントに Key Vault へのアクセス権を付与する

AI エージェントは、実行時に Azure Key Vault から構成シークレットを取得します。 このタスクでは、Key Vault シークレット ユーザー ロールをエージェントのサービス プリンシパルに割り当てることで、シークレットに変更を加えることなく、シークレットへの読み取り専用アクセス権を付与します。

1. **rg-foundry-lab** リソース グループに戻ります。

1. **[Key Vault]** リソースを選択します。

1. **[アクセスの制御 (IAM)]** を選択してから、**[+ 追加]** > **[ロールの割り当ての追加]** の順に選択します。

1. **[ロールの割り当ての追加]** ページの **[ジョブ関数ロール]** タブで、`Key Vault Secrets User` を検索して選択します。

1. [**次へ**] を選択します。

1. **[メンバー]** タブで、**[ユーザー、グループ、またはサービス プリンシパル]** オプションを選択してから、**[+ メンバーの選択]** を選択します。

1. **[メンバーの選択]** ポップアップで、`Contoso-AI-Agent` を検索し、アプリケーションを選択してから、**[選択]** を選択します。

1. **[次へ]** を選択し、**[確認と割り当て]** タブで、**[確認と割り当て]** を選択します。

1. ロールの割り当てが正常に追加されたことを示す通知が表示されます。

AI エージェント ID に Azure Key Vault 内のシークレットへの読み取りアクセス権が正常に付与されました。
